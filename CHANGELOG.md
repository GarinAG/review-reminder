# Changelog

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

## [Unreleased]

## [0.3.0] - 2026-07-06

### Added

- Возможность вернуть МР из игнорирования обратно в очередь (по аналогии с «Проверено»): отдельная секция «ИГНОРИРУЮТСЯ» в меню-баре с кнопкой возврата
- В контекстном меню approve МР прямо из приложения (через GitLab API) с подтверждением

### Changed

- «Отложить → До завтра» теперь откладывает МР до 9:00 следующего дня, а не на 24 часа от текущего момента

### Fixed

- Надёжность обновления меню-бара и цвета в статистике

## [0.2.1] - 2026-07-02

### Added

- Установка через Homebrew: `brew tap garinag/tap && brew install --cask review-reminder` (тап [garinag/homebrew-tap](https://github.com/garinag/homebrew-tap)); cask обновляется автоматически из `release.yml` при каждом релизе

## [0.2.0] - 2026-07-02

### Changed

- Время следующего периодического напоминания вынесено из строки каждого МР в общий футер меню-бара (раньше одно и то же значение дублировалось в каждой строке)

### Fixed

- Периодическое напоминание больше не завязано на цикл сетевого опроса (poll) — расчёт и отправка вынесены в отдельный лёгкий локальный тик (каждые 15 секунд, без обращения к сети), что убирает рассинхрон при равных или близких интервалах опроса и напоминания

### Removed

- Отладочное логирование (`os.Logger`) в `NotificationService`, использовавшееся для диагностики таймера напоминаний

## [0.1.0] - 2026-07-02

### Added

- Уведомление о новом МР (ранее менялся только счётчик в трее, без уведомления)
- Счётчик апрувов (`count/required`) в списке МР
- Счётчик разрешённых обсуждений (`resolved/total`) в списке МР
- Переход к тикету трекера задач из действий над МР: ID тикета парсится из заголовка МР по настраиваемому regex, ссылка формируется из настраиваемого base URL; фича включается/выключается в настройках
- Игнорирование МР по меткам (labels) — настраиваемый список в настройках
- Параллельные запросы к нескольким репозиториям и по МР внутри репозитория (`withTaskGroup`) вместо последовательного опроса
- Глобальный троттлинг запросов к GitLab API (`RequestThrottle`, лимит 5 одновременных запросов) — защищает и поллинг, и запрос статусов в статистике
- Retry с экспоненциальным backoff при 429/5xx от GitLab API
- Частичный успех поллинга: ошибка одного репозитория не отбрасывает уже полученные данные по остальным, ошибка отображается в интерфейсе
- `Taskfile.yml` как замена `Makefile` (на основе [go-task](https://taskfile.dev))
- `README.md`, `CHANGELOG.md`, `.gitignore`
- `LICENSE` (MIT)
- Новая иконка приложения (`Resources/AppIcon.icns`) и `.icon`-бандл для Icon Composer (`Resources/Icon.icon`)
- Скриншоты интерфейса в README (`docs/*.png`)

### Fixed

- Уведомление о периодическом напоминании больше не срабатывает, если все МР уже проверены/заапрувлены — предыдущее запланированное уведомление теперь отменяется, когда очередь опустела
- Периодическое напоминание больше не сбрасывало таймер на каждый опрос (poll) — триггер теперь якорится на время появления самого старого pending МР (`scheduleReminder(anchor:count:afterMinutes:)`) и пересчитывается, только если дата срабатывания реально изменилась
- `UNTimeIntervalNotificationTrigger` заменён на `UNCalendarNotificationTrigger` — напоминание переживает сон/выключение машины; если дедлайн уже прошёл (например, Mac был выключен), уведомление отправляется немедленно при следующем опросе
- `Resources/Info.plist` больше не содержит нерезолвленные плейсхолдеры сборки Xcode (`$(EXECUTABLE_NAME)`, `$(PRODUCT_NAME)`) — они ломали запуск приложения при установке через чистый SwiftPM-путь (`Scripts/install.sh`), минуя `xcodebuild`
- Поля «Ссылка на трекер» и «Regex номера задачи» в настройках выглядели как статичный текст, а не как поля ввода (`TextField` без `.textFieldStyle`, заголовок дублировал значение) — обёрнуты в `LabeledContent` с понятными лейблами и `.roundedBorder`
